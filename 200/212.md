# 212.自制Linux系统镜像dracut篇

Dracut脚本是RedHat Linux系列专属initramfs脚本。

它的核心思想是模块化：它根据当前系统配置，通过执行一系列脚本，将必要的内核模块、二进制文件和库打包成一个压缩的cpio镜像，供内核引导时使用。

本教程使用CentOS Stream 10最新版本作为例子来演示操作过程。内核版本为6.12。

### 一、安装基本镜像

下载CentOS Stream 10的安装ISO，添加到Hyper-V虚拟机里面，使用二代虚拟机、关闭安全启动、关闭检查点执行正常安装。

安装过程中，分区结构选择只分ESP和根分区两个分区，不需要swap分区，根分区文件系统使用ext4，切勿使用lvm等等高级磁盘模式。

CentOS Stream 10 默认会使用lvm进行分区，另外默认使用xfs格式。当然，使用xfs格式也是支持的，为统一和后续操作，建议使用ext4文件格式。


### 二、云固件自举支持程序

云固件相关程序和代码，可以通过在hyperv附加磁盘获取。演示镜像已经提前拷贝到/root目录下。


### 三、准备工作

云固件自举程序包括编译内核模块、编译用户空间程序、添加支持dracut的shell脚本三个部分，需要准备编译环境。

使用root登录，打开终端，进入/root目录

```bash
dnf install kernel-devel
```

输入上述代码安装Linux内核模块开发相关软件包，包含Linux内核头文件以及make、gcc等编译工具，系统会自动选择和当前内核匹配的头文件，无须指定版本。


### 四、编译和安装内核模块

进入/root/下MWVD-6.12文件夹，里面保存了对应6.12内核的分发版本。

```bash
make & make install
```

输入make & make install 命令执行编译和安装。
没有错误的话，mwvd.ko将会被安装到/lib/modules/`uname -r`/updates/下。

编译过程中，由于发行版并不会提供对内核签名的私钥证书，所以会提示签名失败，这个不影响使用。如果需要加固安全，那么可以使用自签名证书添加到系统中，并手动对内核模块进行签名。

由于CentOS 10默认不包含ntfs3内核模块，不支持访问NTFS文件系统，因此需要手动编译添加ntfs3内核模块。

进入/root下ntfs3文件夹，这里保存了Linux 6.12版本的ntfs3源代码。同样，输入make & make install 命令执行编译和安装。

```bash
make & make install
```

编译好的ntfs3.ko模块也会被安装到和mwvd.ko相同的位置。


### 五、编译和安装用户空间程序

```bash
make & make install
```
进入/root/MwSetup文件夹，执行make和make install命令，编译操作内核模块的用户空间程序，成功后会被安装到/usr/sbin/下。

### 六、添加dracut脚本

进入/root/MwBoot/distros/dracut文件夹，这里保存了编辑好的dracut脚本。

复制 90mwvd 到 /lib/dracut/modules.d/下；

```bash
cp -dpR 90mwvd /lib/dracut/modules.d/.
```

复制 mwvd.conf 到 /etc/dracut.conf.d/ 下；

```bash
cp mwvd.conf /etc/dracut.conf.d/.
```

修改/lib/udev/rules.d/60-block.rules, 修改第12行，添加“mwvd*|”到loop*|之后。

修改/lib/udev/rules.d/60-persistent-storage.rules, 修改第10行，添加“mwvd*|”到loop*|之后。

上述2行修改，更新dracut脚本后会被取消。

### 七、生成initramfs启动镜像

上述准备步骤完成后，就可以重新生成initramfs启动镜像了。

```bash
dracut -f
```

执行上述命令，就会重新生成启动镜像。至此，添加云固件镜像自举启动就全部完成了。

### 八、启动测试

关闭虚拟机，复制到需要测试的物理机上就可以测试是否可以通过物理机直接启动。
推荐hyperv结合硬盘透传功能，直接原地测试云固件镜像，不需要复杂的再复制步骤。

注意：由于CentOS默认设置会自动更新内核模块，当出现内核更新后，initramfs会失效，需要重新制作，务必注意。也可以打开grub菜单，启动时选择已经制作好的内核版本，不使用新内核就可以正常启动。
