# 207.自制云固件镜像

云固件通过把操作系统、应用软件等等部署到镜像文件，通过镜像文件实现系统的快速复制、备份、差分等等高级应用。

云固件的镜像使用微软制定的VHDx格式。VHDx格式是微软收购Citrix的Virtual PC后，随Windows 8一起发布的新虚拟磁盘格式。VHDx格式增加了日志系统，并将虚拟磁盘的大小上限扩展到了64TB，同时将区块扩大到了256MB，弥补了VHD最初设计的很多缺陷。

在前面的中级教程中，我们掌握了云固件镜像的基础使用、镜像分层、镜像差分以及镜像备份，把云固件镜像导入到Hyper-V虚拟机里面，还可以虚拟机启动云固件镜像。

云固件的物理机启动虚拟磁盘的技术创新源于Windows 7带来的VHD本地启动（VHD Native Boot）想法。与之不同的是，云固件做了不少创新、增强和扩展。

**一、云固件镜像的增强**

云固件镜像和VHD本地启动镜像，两者尽管都使用了VHD或者VHDx镜像格式，但是两者的结构存在很大的区别。

**1.1 微软VHD本地启动镜像**

在微软官方文档里面可以看到，VHD本地启动镜像只创建单个分区，并通过dism将wim镜像释放到这个分区里面。

参考来源：

https://learn.microsoft.com/zh-cn/windows-hardware/manufacture/desktop/boot-to-vhd--native-boot--add-a-virtual-hard-disk-to-the-boot-menu?view=windows-11

通过制作过程，可以了解到**VHD本地启动镜像不同于正常的操作系统分区，缺少了ESP引导分区**。

因此，这样的镜像是无法在虚拟机里面启动的，只能通过Windows boot manager启动。同时，目前有迹象表明，微软将可能放弃VHD本地启动的支持，VHD系统无法进行大版本升级。

**1.2 云固件镜像**

有鉴于此，云固件镜像在设计初就放弃了VHD本地启动镜像的结构设计，采用了完整的虚拟机硬盘镜像。

云固件镜像不但可以通过Windows boot manager的VHD本地启动能力进行启动，而且可以在支持VHDx虚拟磁盘格式的虚拟机里面进行启动，比如Hyper-V虚拟机。

由于VHD本地启动镜像和云固件镜像都使用了微软VHDx格式，但是内部结构不同。**云固件镜像通常使用“VHDx”来说明格式，与微软的VHDX名称进行区别。**

**二、镜像制作方法综述**

微软在Windows 8之后，把VHDx虚拟磁盘的使用能力整合到文件资源管理器了。在文件资源管理器内，可以直接“加载”VHDx镜像，镜像文件就被当作硬盘呈现出来。

VHD/VHDx文件的应用程序接口，微软也公开了，第三方的应用软件可以直接解析虚拟磁盘文件。

Windows企业版和专业版内置的Hyper-V虚拟机，也默认使用了VHD/VHDx文件格式作为虚拟磁盘格式。

通过上面的信息，我们非常容易得到云固件镜像的几种常见制作方法：

- 通过虚拟机
- 通过调用VHD/VHDx应用程序接口的第三方软件
- 通过复制硬盘

除了上面的普通方法制作云固件镜像以外，还可以借助Windows加载镜像后得到的硬盘，利用其他不支持VHD/VHDx的工具也可以制作云固件镜像。

本篇教程对制作方法进行简单思路说明，在后续的教程中间提供完整的操作步骤和图示。

**三、通过虚拟机制作云固件镜像**

Hyper-V虚拟机默认使用VHD/VHDx文件格式作为虚拟磁盘格式，直接在虚拟机里面安装操作系统，安装完成后关闭虚拟机就直接得到VHDx镜像文件。

由于Hyper-V虚拟机支持名为“检查点”的快照能力，使用此能力将自动对镜像进行差分，因此务必关闭检查点。

另外，类Unix操作系统对安全启动支持不完善，因此此类操作系统安装时需要关闭安全启动。

在虚拟机内安装得到的VHDx镜像文件，能够满足云固件启动镜像文件的第一阶段要求，但是第二阶段镜像内操作系统启动自身通常不能直接满足。因此，不同的操作系统需要使用不同的后续处理步骤。

比如Windows系统，需要修改注册表，关闭VHD文件自动扩展空间的能力。

比如Linux系统，需要增加内存盘启动脚本，来支持Linux镜像从VHDx镜像文件中间启动。

**四\**、通过第三方软件\****

云固件镜像文件还可以通过第三方的软件进行制作。云固件镜像本身是一个完整的虚拟硬盘。其他虚拟硬盘可以通过格式转换工具来转换生成云固件镜像的VHDx格式。

云固件演练场制作工具中提供的免费软件Starwind Converter能够支持VMware Workstation的VMDK格式转换成VHD/VHDx格式。

开源软件QEMU提供的qemu-img工具可以支持常见的全部虚拟磁盘格式转换成为VHDx格式，这些格式包括RAW、VDI、QCOW2、VMDK、VHD等等。

***\*五、通过复制硬盘\****

云固件镜像作为完整的虚拟硬盘，还可以通过复制硬盘的形式获得VHDx文件。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoTqSCWKa2a0gpja7LlMFicpKA0E9o633R1IAyKv0GmCFdrVabENSwWJrfd3Ksx4icA5wDHAlzGHSPQ/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

Disk2vhd就是微软系统工具箱中间的一个P2V工具，支持将当前系统复制到VHDx文件。

上面提到的Starwind Converter也支持物理硬盘复制到VHDx文件。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaqHVG8rFhM0huk7MRWQqMJlpxicDwtjvoabV3zOaVLd6NiabBebK9ZuiafX1df1b8Hsm6XicU73ibdPOxw/640?wx_fmt=other&from=appmsg&wxfrom=5&wx_lazy=1&wx_co=1&tp=nowebp)

Hyper-V虚拟机管理器也支持从物理硬盘复制成为虚拟磁盘VHDx文件。

我们经常使用的Disk Genius也支持复制硬盘到VHD格式，注意目前DG原生只支持VHD格式，不支持VHDx格式。

以上方法复制得到的硬盘和物理硬盘是完全复制，因此硬盘分区的UUID也是完全相同的。这样得到的镜像在复制主机上使用会导致原本唯一的UUID出现人为的重复，会导致Windows出现蓝屏错误。如果需要本机使用，那么或者清理本机分区信息，或者更改VHDx镜像文件内的分区UUID。

复制硬盘时，除了复制完整硬盘，还可以复制单独分区。

比如，物理硬盘分成ESP、操作系统区、数据区三个分区。那么可以通过DG只复制ESP和操作系统两个分区，不复制数据区。这样得到的虚拟磁盘VHDx文件可以使用不同的分区UUID，不会和本机硬盘冲突。

由于硬盘分区的变更，ESP分区内的引导信息就会出现不一致，为了保证虚拟磁盘能够通过ESP分区引导，需要修复引导配置信息。

**六、通过Windows加载虚拟硬盘和其他工具**

Windows 8之后的各个版本Windows都支持加载VHDx文件为硬盘。加载后得到的硬盘和普通硬盘无异，对于其他应用软件来说，都能读取和写入。

Disk Genius原生不支持VHDx文件，通过这种方式，DG就可以实现完整硬盘复制或者分区复制，从而得到云固件镜像文件。

传统的备份软件，使用自己特殊格式存储的备份文件，也可以通过对应的恢复工具恢复到加载的虚拟硬盘文件上，这样也可以得到备份前的完整硬盘内容。

**七、总结**

虚拟硬盘的读写工具众多，因此制作云固件镜像的方法也是繁多，即使不支持VHDx格式的工具也可以实现镜像制作。后续的教程会针对常见系统提供完整操作实践步骤来完成不同系统的镜像制作，欢迎大家阅读。
