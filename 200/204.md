# 204.分层及差分实践

在之前的两篇教程中讲解了《[【云固件教程】202.镜像差分](http://mp.weixin.qq.com/s?__biz=MzkxNzYwOTk5OQ==&mid=2247484201&idx=1&sn=93a0965209b019354a6c0a295157ced3&chksm=c1bf4849f6c8c15f7da94c87a04a099e4d9b37066abab41c373d868e1d11a24c688a96968963&scene=21#wechat_redirect)》和《[【云固件教程】203.镜像分层与分层规范](http://mp.weixin.qq.com/s?__biz=MzkxNzYwOTk5OQ==&mid=2247484203&idx=1&sn=c383c61148dd0a6cda8d04710f0a4b9c&chksm=c1bf484bf6c8c15d41e9ce63e5a5fed6977bc44cabd38b935b5a251223876315d11b915af8ec&scene=21#wechat_redirect)》。镜像分层与差分，目的是为了建设一个镜像文件体系，将重复劳动分隔开，避免重复再劳动。

镜像文件体系的建立不是强制要求，属于推荐性的实践方案，在不同场景下合理使用分层和差分能够有助于提高问题解决效率。

本篇教程提供多个完整的实践用例，来帮助大家理解和消化分层和差分在实际场景中的使用。

**场景一**

隔壁老王家有一台计算机，老王用来学习Python开发，老王爱人用来做电子表格，老王儿子小王用来打游戏。

最初，老王、老王爱人、小王都是用同一个默认账户登录，自己装自己用的软件或者游戏，然后使用。

后来，小王在安装游戏过程中，删除了部分VC runtime library，导致了老王的学习环境和老王爱人的电子表格程序无法使用。老王和老王爱人很生气，就给小王单独设置了一个用户，让小王在自己的用户下安装和运行游戏。

再后来，小王又装了一个盗版游戏，中了木马，计算机成了肉鸡，不仅小王自己的账户无法使用，连带老王和老王爱人的账户也完蛋了。

现在使用云固件后，虽然是在同一台计算机上，但是老王、老王爱人、小王通过差分得到了三个操作系统，分别是Code、Work、Game。这三个系统虽然源于同一个镜像进行差分，但是三个系统是完全独立的，任意一个系统出现问题，删除子镜像，重新差分即可复原使用环境。

配置文件告诉我们，三个入口子镜像分别是Game、Work和Code。

再看一下镜像文件夹内文件构成。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7Ay9ep75dlJBzWEJxYvQXiaorxjRaXQ02stgJCUPugLXcAVI9WGmZtpfgg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

我们可以看到三个子镜像和父镜像“Windows-11-LTSC-2024-NUC11PA-L5.vhdx”。

Game.vhdx、Work.vhdx、Code.vhdx都是通过差分“Windows-11-LTSC-2024-NUC11PA-L5.vhdx”得到。这是差分的使用。

父镜像的文件名告诉我们，这是一个Windows 11的镜像，版本为LTSC 2024版本，针对的主机型号是NUC11PA，也就是Intel NUC 11代猎豹峡谷，镜像层级为L5层，说明了已经安装了驱动、更新了升级补丁、激活了系统、并且可能安装了部分常用软件工具比如7-zip等等。这是分层的使用。

老王家这台计算机的案例是非常通用的案例，单一用户也会存在这样的需求，也就是这个用户有多种使用场景的需求，包括游戏、简单办公、学习等等，可以通过在主镜像上差分获得不同环境来实现。主镜像在L1层级的基础上，不断补充不同层级的内容，最终到达L5层级，获得了主镜像。备份好主镜像，后续不同环境都可以在主镜像上差分获得。

**场景二**

本场景基本需求同场景一，唯一不同的地方是老王最近迷上了AI开发，所以显卡驱动使用了不同于游戏场景的驱动（举例，不一定真实）。

现在全部子镜像通过L5主镜像来实现，有些不太方便了。

咱们先来看menu镜像配置文件。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7AyBINbPGrY4Nd8DJUMMcH96ffwf3q87viaMrAQ2B2HBA7opiaqibMjsyWPA/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

配置文件告诉我们，三个入口子镜像分别是Game、Work和Code。这个情况和场景一完全相同，看不出来差别（基础镜像的Windows 10和11差别不算，只是举例不同）。

咱们来看镜像文件夹的构成。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7AyfvlI8ZzHqzk5t8FNMVT1hz6bjS5fwV2ibh1NrZUW5CfblMzOVRBiatPQ/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)



可以看到Game.vhdx、Work.vhdx、Code.vhdx三个入口子镜像。但是主镜像发现有L1、L3、L4三个。

这里的分层和差分结构就相对复杂了。

首先，L3镜像是在L1镜像上差分得到，是L2和L3两层的叠加，也就是安装了全部驱动程序、完成了更新。但是，这些操作不同于场景一，没有叠加到L1层的镜像文件上。L4层镜像是在L3层镜像上继续差分得到，在这个镜像文件上实现了系统激活。

其次，实际使用环境的Work和Game是在L4层级镜像上差分得到，分别用于工作和游戏。

最后，是复杂的Code子镜像。由于老王需要修改后的显卡驱动，显然在L2以后级别镜像基础上生成子镜像是比较麻烦的，所以老王直接在L1级别上差分得到了Code子镜像，并在Code子镜像内安装了修改过的显卡驱动，获得了他需要的特殊环境。当然，老王其实可以在安装了修改过的驱动之后保存一个特别的L2镜像，然后继续差分进行后续其他软件包的安装。

在这个场景中，主镜像不唯一，镜像分层和差分进行了交叉，满足了同一显卡安装不同驱动的特殊需求。

**场景三**

隔壁老王在一家LED室外屏公司上班，编写控制LED室外广告屏的应用软件。广告屏通过硬件接口控制，需要部分驱动程序的调试和测试。不同型号的驱动还有所不同，为了节约成本，设备编号使用了相同的编号。安装时，需要自己选择不同的驱动程序。

这里的需求明显要复杂了，咱们直接看镜像文件夹。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7Ayx5P5Rf2uSnxBnsVWeYB9zkTdnRGtZnibWCSibfaSicxduQ5AiarL4HIZcg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

首先，我们可以注意到镜像文件突然变得非常多。仔细观察，发现主镜像是从L1、L2、L3、L4都齐全了，也就是说每层都保存了独立的差分镜像，没有叠加合并。

其次，在分层镜像上差分得到了多个子镜像，比如在L3层上差分得到了301，L5层上得到了500，501、502、503。

最后，镜像配置文件还存在两个。咱们分别来看一下内容。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7AySG3JRBic8iafbTgKPcCq2AqBUzcumxzRtIZHtsIcOh2EE6W4qHPX1iaQg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAaoHNZEYPhtPclBZSJrlm7AyBt8icNaSibicT59uSXcjLZHjSUnvchJBL8Qic9RzDIibqqfp8Nn5hknOIiag/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

可以看到301、501、502、503分别是最终入口的子镜像文件。由此可推断301子镜像可能是单独安装另外一个版本驱动并更新后的镜像。剩下的501、502、503可能是基于同一个版本驱动后配合不同测试版本应用软件的子镜像。

那500镜像是什么呢？由于测试版本或者分支版本较多，需要不断重复差分L4镜像得到L5层级镜像，老王直接把差分后的子镜像留存了一份，需要使用的时候直接复制这个子镜像，然后更名就可以直接使用了，不再使用差分工具。

在这个场景中，存在多个层级不同分支的差分子镜像，同时也需要快速得到差分镜像。尽管与前面的场景有明显不同之处，但是依旧可以通过分层和差分得到妥善的解决。

**总结**

镜像分层和镜像差分是两个相互组合使用的方法，合理组合能够达成很多原先非常不好处理的使用需求。但是，过多的分层也会影响使用效率，可以通过合理的合并层级来优化使用效率。差分子镜像同样可以差分获得，也可以复制子镜像获得。方式和方法都很灵活，最佳使用方式可以优化再优化。对于特定场合，也可以不考虑使用差分和分层，适合才是最优的方式。
