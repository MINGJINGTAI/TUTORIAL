**二、镜像文件与镜像文件夹**

在102教程中，对镜像文件和镜像文件夹做了简单的说明，咱们来复习一下。

云固件镜像并不单独使用VHD(x)文件，而是使用镜像文件夹来保存镜像相关的文件，镜像相关文件包括镜像文件、该镜像的菜单ICON文件、镜像配置文件、镜像背景图片（效率问题暂未支持），以及其他说明文件或者镜像启动辅助文件。

保存镜像的镜像文件夹命名是有要求的，通过分享和下载拿到的镜像文件夹的名称是不能更改的。如果需要更改，那么就需要修改配套的配置文件，否则云固件主程序将无法找到并启动镜像文件。在不了解配置文件构成前，不建议做任何改动。

云固件通过镜像文件夹来管理镜像文件和相关资源文件。

大家学习时，多数情况就会下载1-2个镜像文件夹。但是小助手却维护着大概超过600G的镜像文件夹数据，还都是压缩文件的形式，平均每个镜像文件夹在7.5G左右，可以计算出大概是80个镜像文件夹，未来还在不断添加中。这80个文件夹怎么命名就是一道难题，像Win10Pro这样的命名显然会重复。

其次，大家也不断询问menu.config中间UUID的含义，因为UUID的头8个字符就是目前使用的文件夹名称。为了避免上面重名的情况，所以选择了使用UUID来区分，这样重名问题就解决了。同时，选择UUID也是为了接下来v2.x版本中间镜像商店的需要，镜像商店面临的镜像标识问题更加突出，数量很有可能突破万，甚至十万、百万，那怎么标识，显然是唯一ID，也就是UUID了。

以上是官方标准解释。

**三、云固件演练场**

制作工具文件夹“99887766”本名叫做《云固件演练场》，是给云固件使用者提供的操作环境，也可以叫做练手环境、练习场，相当于游戏中的“新手村”。但是，演练场的能力远远不是新手练习这么简单。

云固件镜像在最初设计中是当作资源文件来提供的，资源文件需要应用到云固件使用环境中才能使用。

显然，中间需要操作步骤的。但是，对于初学者来说，多一个步骤，失败概率就会多一点，放弃的概率就会大一点。

因此，经过慎重考虑后，最初的教程里面就压根儿不提使用的操作过程，直接按照镜像操作三步骤来使用。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAarre0iak5jLrqflGv6IX2mkCgrDkK9eCHMice5yic4JPSnGaiaZ52XSEfQCPxCQbGUx07j0AXnSXlYs6w/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

实际上，云固件官方发行的磁盘镜像分成两种类型：需配置后使用及直接使用。

- **配置后使用的镜像**

通常为分层规范中的中间格式，提供该镜像的目的是给使用者在此基础上进一步部署后使用。配合云固件镜像演练场，使用者可以通过复制或者差分该基础镜像进行后续加工处理。（差分及分层参考中级教程202、203）

- **直接使用的镜像**

通常为制作完毕的镜像，制作者提供了差分好的子镜像，并添加了menu.config中的引导参数。使用者只需引入全局vd.config即可在启动界面上看到对应的选择菜单。当然，使用者还可以继续创建差分或者进一步加工到自己需要的更高层级使用环境。

看上面的说明，第二种类型“直接使用的镜像”对初学者来说更加方便，所以前面的教程就推荐了这种方式，把后面加工的步骤也直接忽略了。

推荐的镜像使用方式，需要使用者复制或者差分原始镜像来使用，其中修改镜像名称是必然的步骤，默认的官方镜像名称绝大多数都是L1层级镜像名称，既不符合对应的层级，也不符合使用的机型特征。

不管是差分还是复制，显然需要目标文件夹，云固件演练场就是为了这个目标而诞生的，把需要的镜像放到演练场里来使用，同时演练场也为接收测试镜像做了准备工作。

首先，默认的vd.config配置已经包含了演练场的引入设置；

其次，menu.config已经做了多数配置项，只需要简单修改镜像文件名即可完成配置；

```
menuentry "AINUC Multiware Playground" {  
icon "\99887766\win11.png"  
vdisk "\99887766\Work.vhdx"    
vdtype vhdx    
ostype windows
}
```

最后，绝大多数工具都在这个文件夹提供了，都是安全无广告的官方版本。



**四、云固件镜像的标准使用方式**

不管是配置后使用的镜像还是直接使用的镜像，小助手都建议复制到演练场里面来使用。这样，原始镜像文件夹可以保持有序和未更改，便于后期再利用。

- **复制模式**

将镜像文件夹内的镜像文件复制到演练场，更名为符合自己主机和实际分层的镜像文件名称；同时复制相关的资源文件，包括图标、背景、辅助启动文件等等；对比配置文件，如有特殊配置也要复制到menu.config，并更改配置文件中的镜像文件名称，与实际对应上。

- **差分模式**

使用差分时，Windows在父子镜像文件跨文件夹时存在问题，需要把父子镜像都放在相同文件夹下。Windows下可使用演练场内提供额Bootice来完成。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapFMvhNpMfeXYDjoxXLMribVuqpDWHJcw9G3mtVV0XZETSv7cdJCfqiagrW9gvFedpkGpgtWlbUCHvg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

创建差分后，也要对比配置文件，如有特殊配置也要复制到menu.config，并更改配置文件中的镜像文件名称，与实际对应上。更多差分操作，可以参考教程202。

Linux等系统不支持差分，只支持复制模式。

**五、****总结**

云固件及云固件镜像属于一种创新的计算机系统部署模式，很多方法与传统方法都有很大的差异，也不能用传统方式来考虑使用方法。演练场为大家使用云固件镜像提供了一个相对简单的环境，让学习者更容易上手。单个镜像的使用方法可以参考教程《204.差分与分层实践》，多个镜像组成的应用环境可以参考教程《205.应用环境概念与实践》。
