# 108.云固件混合模式（与本地系统混合启动）

**一、前言**

云固件支持U盘的外置模式、移动存储的移动模式、独占主机的独占模式（标准安装模式），但这三种模式都有所不足，或不便，或无法并用。云固件的混合模式将云固件和本来的系统混合在一起使用，提供了更全面的使用模式。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAar7GibOe13QgqyZ3Ao5VAjtiaG9IqEr77pxUibASyAVWfzicA0yZzBI7bWF5Z7wxcKGb2UYBmIzrjHInQ/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

云固件的混合模式有很多前提条件：

- 系统必须是UEFI模式进行启动的
- 内置硬盘必须是GPT模式进行的分区
- 内置硬盘不能加密
- 内置硬盘不能有加密或者压缩的分区
- 主机必须关闭安全启动（Secure Boot）

不能满足以上全部条件，混合模式是无法部署和使用的。

**二、安装步骤之复制文件**

继续使用之前的制作工具DG和云固件主程序v1.6.23942版本。云固件所有模式使用的主程序和镜像文件都是一样的，没有因为模式不同而选择不同的版本！

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5qRhNNhjmlAv0DrzjQsOzttG07yNAy6ao7kOX1NTJzhDd4BUWeMGLibw/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

将红色框的Drivers和MW两个目录拖入内置硬盘的ESP分区下的EFI目录。

如果你看不到你主机上的ESP分区或者EFI目录，那么你的主机不是GPT+UEFI模式，不满足安装条件。

***注意：不再是把云固件主程序的EFI目录全部使用了，只要其中的两个文件夹了。***

**三、安装步骤之添加启动项**

云固件主程序添加到ESP分区后，不再是默认启动项，所以需要手动添加启动项。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5jXSfeJMIibH19T5JFICchU4Pe6Vawq80BSDcuibJjHTUvngbictsTzljg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

如图，选中“UEFI BIOS启动项”

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5d3MEjxibvG4BsYcDAB3wcbUwia5QNIMzdKS5KgKJEmQiawJBxOSPPqj0A/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

显示如上画面，选择“添加”

弹出如下画面，搜索到ESP分区下EFI\MW\MW.efi，点击右键添加到下方。***注意，必须选中之后右键才能添加，不是双击。\***

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5xuAVdMv7AzGbknkPekCHfic6Qhu3Vr76Iaj4iaxpCslKlaj8aWUGJbDw/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt56lQ7SXmte9hp4z7aB3rwpvUKklN6MEvkvibhcibxgWzseicK5lYaJtA4w/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

添加后，可以修改提示标题，云固件英文为“Multiware”，所以设置为“Multiware Boot Manager”，名称可更改，但不能出现中文和特殊字符。

然后点击保存，之后即可退出DG。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5tQmGzBtAbzNrTE8cW96kvw7yjichttCiaHoKQxyqFjPDGkONv9ATBrfA/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

通过以上步骤，添加启动项完成。

**五、本机系统整合**

虽然云固件的启动再也不需要启动U盘或者移动存储了，但是每次启动系统时都需要通过BIOS的多重启动选择一下，比较麻烦，是否可以把本机系统整合到云固件的启动界面上呢？

当然是可以的，参考以下操作。

首先，编写如下内容的vd.config文件，只适合Windows为本机主系统。

- 
- 
- 
- 

```
menuentry "Local Windows" {  icon "\EFI\MW\icons\os_win.png"  loader "\EFI\MICROSOFT\BOOT\BOOTMGFW.EFI"}
```

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5xdgAzWVp8M1rFeRRnDpiafHBUsGMKcZuCHXwbvmetqVmX4w6GeVsFaw/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

将此文件拖拽到ESP分区的根目录下

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5ZapyEqZg4bLvS2xV7tfU5iaEgKzv2GpYmvEdibKrmxrNEDZJ5TSFPnjg/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

然后重新启动计算机，继续选择Multiware Boot Manager。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5nWr9Y9fuU1Qo5zT2jX3pzICYGBndfJZyG9NKIg7LGUX03XsjCkjklQ/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

这时候就可以看到本机的Windows就显示出来图标了。

当然，这时候还是需要选择多重启动，是否可以默认就是云固件启动呢？那也必须是可以。参照BIOS里面的启动优先级来调整。下图是样机上的顺序调整界面。

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5GMMwK0x71ibLu7bpHliaDHHnicG9ZtKIHq2At7xzt9MLcaz0P3sHy8bjA/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

调整前

![图片](https://mmbiz.qpic.cn/mmbiz_png/sgxgSdyNAapXjev0ZKZXGxRqNUn6cxt5sKRWicU5UmvJFep7r5pzoSTSbq5p5ibuqbGWUwcSQia5zQNCqr404yn0A/640?wx_fmt=png&from=appmsg&tp=nowebp&wxfrom=5&wx_lazy=1&wx_co=1)

调整后，Multiware就是第一个启动选项了，开机就会直接进入云固件主界面了。

**六、****总结**

云固件的混合模式需要对原系统的ESP分区进行修改，任何步骤错误或者误操作其他文件，都可能造成本机系统无法启动的风险，所以没看明白之前切勿操作，防止出现不可挽回的意外。
